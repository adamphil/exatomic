Cube File Support¶

Cube files contain an atomic geometry and scalar field values corresponding to a physical quantity.

class exatomic.filetypes.cube.Cube(*args, label=None, field_type=None, **kwargs)[source]¶

An editor for handling cube files.

cube = Cube('my.cube')
cube.atom                # Displays the atom dataframe
cube.field               # Displays the field dataframe
cube.field_values        # Displays the list of field values
uni = cube.to_universe() # Converts the cube file editor to a universe
uni                      # Renders the cube file

Warning

Be sure your cube is in atomic units.

parse_atom()[source]¶

Parse the Atom object from the cube file in place.

parse_field()[source]¶

Parse the scalar field into a trait aware object.

Note

The AtomicField object tracks both the field data (i.e. information about the discretization and shape of the field’s spatial points) as well as the field values (at each of those points in space).

exatomic.filetypes.cube.write_cube(path, universe, frame=None, field=None)[source]¶

Generate a cube file or files from a given unvierse.

Only the path and universe are required; if no frame or field indices are provided, this function will write all fields in the given universe. If multiple frames are desired, pass a list of frame indices; the field argument should then be a list of the same length with the corresponding field indices or none (in which case the first field for each frame will be written). If multple fields per frame (multiple) is desired, use a list for frames as previously, but use a list of lists for fields where the outer list has the same length as the frame argument list and each “sublist” now contains the indices of the fields specific to that frame to be written.

Parameters:
  • path (str) – Directory or file path
  • universe (Universe) – Universe to pull data from
  • frame (list or int) – If only a specific frame is needed
  • field (list or int) – If only a specific field is needed

Note

“Indices” in the above description always refer to dataframe index. Also always writes in atomic units.
